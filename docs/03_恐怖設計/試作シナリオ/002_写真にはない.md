# 試作シナリオ：写真には、ない

## 基本情報

- ID：SCENARIO-002
- 想定プレイ時間：20〜30分
- 想定総選択回数：20〜24回
- 重要な選択：9〜11回
- 小さな選択：10〜13回
- 使用する怪異：`ENTITY-002 欠落`
- 使用する主法則：`RULE-002 存在を否定してはいけない`
- 使用する副法則：写真を見る前に現実側の痕跡を確認すると、消去を遅らせられる
- 難易度：中

---

## このシナリオで検証すること

- 追跡や回数ではなく、認識と言葉を使った攻略が成立するか
- 20回以上の選択でも、一つ一つの役割を維持できるか
- 小さな探索選択と重要な法則選択を組み合わせられるか
- 写真と現実の不一致を、単なる間違い探し以上の恐怖にできるか
- 過去ログの改変演出が、理不尽ではなく恐怖として機能するか
- 最終局面でプレイヤー自身の存在を問う展開が成立するか

---

## 最初の3つの物

```text
受付台の上に、三つの物がある。

古いインスタントカメラ
名前の削られた真鍮の鍵
何も録音されていないカセットテープ

ひとつだけ持っていける。
```

### 古いインスタントカメラを選んだ場合

本シナリオへ進む。

カメラは、現実が書き換わる直前の状態を一度だけ記録できる。

ただし、撮影した写真にも怪異は干渉する。

### ほかの物を選んだ場合

試作段階では別シナリオへの分岐想定のみ残す。

---

## 今回の舞台

- 場所：閉店した古い写真館
- 時間：外の時間が分からない
- 主な空間：
  - 受付
  - 待合室
  - 撮影室
  - 現像室
  - 写真保管庫
  - 裏口へ続く廊下
- 脱出目標：写真に写らなくなった裏口を、現実側の痕跡を頼りに開ける

AIは写真館の名称、写真の年代、家具や照明の細部、記録文の言い回しを変えてよい。

---

## 固定する世界設定

- 写真は現実より一段階先に書き換わる
- プレイヤーが欠落を直接名指しすると、現実側にも消去が定着する
- 対象の周囲に残った痕跡は、すぐには消えない
- 現実を先に確認し、痕跡を記録すると消去を遅らせられる
- 裏口は中盤以降、写真から消える
- プレイヤーが裏口の消失を確定すると、現実の裏口も消える
- 最終写真には、撮影したプレイヤーの存在を示す影がない
- プレイヤー自身の欠落を認めるとゲームオーバーになる
- 正しい脱出方法は、写真ではなく蝶番・床の擦れ・冷気など現実の痕跡を信じること

---

## 内部状態

| 状態 | 初期値 | 更新条件 | 用途 |
|---|---:|---|---|
| `entity_pressure` | 0 | 名指し、写真優先、空欄入力で上昇 | 演出と侵入判定 |
| `named_absence_count` | 0 | 欠落対象を直接名指しすると+1 | 現実消去とゲームオーバー |
| `reality_check_count` | 0 | 写真を見る前に現実を確認すると+1 | 消去遅延と選択肢解放 |
| `anchors` | 空 | 痕跡を記録すると追加 | 最終脱出判定 |
| `erased_objects` | 空 | 名指しした対象を追加 | シーンと移動の変化 |
| `photo_uses` | 0 | インスタントカメラ撮影で+1 | 証拠取得と副作用 |
| `self_erasure` | false | 自分の欠落を認めるとtrue | 即ゲームオーバー候補 |

怪異圧力は0〜5の段階で管理する。

```text
0：未認識
1：写真の違和感
2：記録の欠落
3：現実の消失
4：ログの改変
5：プレイヤー自身の欠落
```

# クリア条件

基本クリア：

- `self_erasure == false`
- 裏口を「存在しない」と確定していない
- `EXIT_TRACE`または同等の出口アンカーを取得している
- 最終局面で写真ではなく現実の痕跡を選ぶ
- 退店者入力で、自分の存在を怪異へ渡さない

クリアに必要なアンカー数は、MVPでは2つ以上を目安とする。

ただし、出口に関するアンカーは最低1つ必要とする。

---

# ゲームオーバー条件

### 欠落の確定

脱出に必要な裏口を直接否定し、現実から消去させる。

### プレイヤー自身の否定

「撮影者はいない」「自分は写っていない」などを確定する。

### 名指しの蓄積

`named_absence_count`が規定値に達し、怪異圧力が5になる。

### 最終入力の失敗

高い怪異圧力の状態で自分の名前を入力する。

# エンディング案

## 基本生還

写真を信じず、現実の痕跡を頼りに裏口から出る。

## 不完全生還

自分の名前を入力して脱出する。

外には出られるが、その後の記録に名前が残らない。

## 真相到達

自分の欠落を示す写真を見ても、身体と影を確認し、写真を伏せて脱出する。

## 同伴エンド

退店者を「ひとり」と入力し、写真館内の何かを人数に含めて外へ出す。

# プレイテスト確認項目

- 椅子の場面で法則の方向性を感じられるか
- 「欠落を名指しする」と「痕跡を記録する」の違いが伝わるか
- 20回以上の選択が冗長にならないか
- 小さな選択に探索や理解の意味があるか
- 裏口の消失が、それまでの行動につながっているか
- ログ改変が混乱だけでなく恐怖として機能するか
- 自分自身の欠落が唐突に感じられないか
- 名指し違反を一度しただけで詰まないか
- ゲームオーバー後に過去ログを確認したくなるか

# 試作段階での割り切り

本シナリオでは、AIが自由な文章入力を意味解析して法則違反を判定する仕組みは使わない。

すべての選択肢に内部IDを持たせ、

```text
DIRECT_NAMING
TRACE_RECORDING
REALITY_CHECK
BLANK_SUBMIT
SELF_DENIAL
```

などの分類で判定する。

これにより、AIの文章表現が変化しても、ゲームルールの一貫性を維持する。

最初の実装試作では、全21選択を実装せず、主要な流れだけを10〜12選択へ圧縮して動作確認してもよい。

ただし、完成版の深さを検証するため、文章上のシナリオ設計は20回以上の選択を前提として残す。
