# 試作シナリオ：選ばなかった方

## 基本情報

- ID：SCENARIO-003
- 想定プレイ時間：20〜30分
- 想定総選択回数：21〜25回
- 重要な選択：10〜12回
- 小さな選択：11〜13回
- 使用する怪異：`ENTITY-003 残された方`
- 使用する主法則：`RULE-003 捨てた選択肢を選び直してはいけない`
- 使用する副法則：自分が選んだ道には、片道切符で印を残す
- 難易度：中〜高

---

## このシナリオで検証すること

- 選択肢そのものを恐怖演出へ利用できるか
- 過去ログが攻略に必要な情報として機能するか
- 未選択肢を別の主体として進行させられるか
- 20回以上の選択履歴を、プレイヤーが追えるか
- 選択肢の時制・位置・句読点による状態示唆が有効か
- プレイヤーの選択傾向を、心理的な揺さぶりへ利用できるか
- 「選択すること自体が怖い」というコア体験を直接表現できるか

---

## 最初の3つの物

```text
券売機の受取口に、
三つの物が残っている。

裏面のない片道切符
半分に割れた白いコイン
右手だけの黒い手袋

ひとつだけ持っていける。
```

### 裏面のない片道切符を選んだ場合

本シナリオへ進む。

切符は一度選んだ経路へ印を残し、その経路が「自分が選んだ側」であることを証明する。

ただし、使える回数には限りがある。

### ほかの物を選んだ場合

試作段階では別シナリオへの分岐想定のみ残す。

---

## 今回の舞台

- 場所：営業を停止した地下鉄駅の連絡通路
- 時間：終電後に見えるが、時計は動いていない
- 主な空間：
  - 改札前
  - 左右に分かれる連絡通路
  - 無人ホーム
  - 非常階段
  - 保守用通路
  - 出口表示のない地上階段
- 脱出目標：自分が実際に選んできた経路を維持し、地上へ出る

AIは駅名、案内表示、広告、足音、照明などの細部を変えてよい。

---

## 固定する世界設定

- プレイヤーが選ばなかった行動も、別のルートで実行され続ける
- 各場面の未選択肢は内部状態として保存される
- 未選択肢は数場面後、現在の選択肢へ戻ってくる
- 再出現した未選択肢には、時制・句読点・番号・表示タイミングのいずれかに兆候がある
- 未選択肢を選び直すと、別の主体の完成度が上がる
- 別の主体は、プレイヤーが捨てた行動だけで構成される
- 片道切符は、自分が実際に選んだ経路を三回まで記録できる
- 最終局面では、選んだ側と選ばなかった側の二つの履歴が提示される
- 自分が選んだ履歴を維持したまま出口を選ぶと生還できる

---

## 内部状態

| 状態 | 初期値 | 更新条件 | 用途 |
|---|---:|---|---|
| `chosen_history` | 空 | 各選択で選択肢IDを追加 | 自分のルート判定 |
| `abandoned_history` | 空 | 各選択で未選択肢IDを追加 | 再出現候補 |
| `returned_options` | 空 | 未選択肢を現在へ戻す | 法則確認 |
| `echo_selected_count` | 0 | 戻った未選択肢を選ぶと+1 | 別の主体の完成度 |
| `other_self_level` | 0 | 違反・覗き見・偏りで上昇 | 演出と終盤判定 |
| `ticket_marks` | 3 | 切符で経路を記録すると-1 | 自分のルート証明 |
| `route_anchors` | 空 | 印を残した経路を追加 | 最終クリア判定 |
| `position_bias` | なし | 同じ位置を続けて選ぶと更新 | 選択傾向演出 |
| `identity_swapped` | false | 最終重大違反でtrue | ゲームオーバー |

`other_self_level`は0〜5で管理する。

```text
0：未形成
1：足音
2：行動の断片
3：文章への侵入
4：履歴の入れ替え
5：現在への到達
```

## フェーズ5：最終局面

### シーン18：二つの履歴

出口前の端末に二つの記録が表示される。

```text
あなたが選んだ行動

あなたが選ばなかった行動
```

切符の印を残した経路だけは判別できる。

#### 選択19・重要

1. RECORD Aを自分の履歴として選ぶ

3. 切符の印だけを確認する
4. どちらも自分だと認める

3を先に選ぶと、正しい履歴を判別しやすくなる。

4は統合エンド候補。

4. 選ばなかった方を待つ

1は一見正しそうだが、履歴が大きく汚染されている場合は危険。

2はゲームオーバーまたは交代エンド。

# クリア条件

基本生還：

- `identity_swapped == false`
- 戻った未選択肢を重大回数以上選び直していない
- `route_anchors`を1つ以上保持している
- 最終局面で、過去形の未選択肢を選ばない
- 自分の履歴を、切符または固定された警告によって維持する

安定した生還には、切符の印を2つ以上残していることを推奨する。

# ゲームオーバー条件

### 未選択肢の選び直しの蓄積

`echo_selected_count`が規定値に達し、`other_self_level == 5`になる。

### 履歴の交代

最終局面でRECORD Bを自分の履歴として確定する。

### 選ばなかった方への譲渡

最後の問いで「選ばなかった方」を選ぶ。

### 現在の放棄

「選ばなかった方を待つ」を選び、現在の経路を手放す。

# エンディング案

## 基本生還

切符の印を頼りに、自分が実際に選んだルートを維持して地上へ出る。

## 不完全生還

自分の履歴を選んで脱出するが、過去ログの一部に別の自分の選択が残る。

## 統合エンド

「どちらも自分だ」と認める。

脱出はできるが、その後の選択肢が常に二重表示される。

## 交代エンド

選ばなかった方を自分として選び、現在のプレイヤーがログ側へ残される。

# 状態示唆の軽い演出

`other_self_level`に応じて、以下を段階的に使用する。

### レベル1

- 未選択肢が消えるまで一瞬遅れる
- 過去ログに空行が残る

### レベル2

- 未選択肢が過去形で戻る
- 選択カーソルが一瞬だけ未選択肢へ移動する

### レベル3

- 三択目として未選択肢が混ざる
- 選択していない結果が一行表示される

### レベル4

- 過去ログの選択済みマークが入れ替わる
- 選択肢番号が不規則になる
- 「いつもの方を選ぶ」が現れる

### レベル5

- 選択肢が一瞬二重に表示される
- 自分の選択より先に、未選択側の結果が表示される
- タイピング音が左右から交互に聞こえる

数値ゲージは表示しない。

# プレイテスト確認項目

- 初めて戻った未選択肢を判別できるか
- 時制の違いが露骨すぎないか、分かりにくすぎないか
- 過去ログを確認する行為が面倒ではなく攻略として機能するか
- 21回前後の選択履歴を追えるか
- 小さな選択と重要な選択の比率が適切か
- 切符の印が強すぎて法則が簡単にならないか
- 選択位置の癖を利用した演出が不快な罠になっていないか
- 未選択肢を一度選んだだけで詰まないか
- 最終局面で自分の履歴を選ぶ意味が理解できるか
- ゲームオーバー後にログを比較したくなるか
- 「選択すること自体が怖い」という感覚が生まれるか

# 試作段階での割り切り

再出現する未選択肢は、AIが意味から自由に選ぶのではなく、ゲーム側が選択肢IDで指定する。

例：

```text
CURRENT_OPTION
ABANDONED_OPTION_RETURN
ROUTE_ANCHOR
IDENTITY_SURRENDER
```

AIは表示文を変えてよいが、内部分類を変更してはいけない。

最初の実装試作では、全21選択を10〜12選択へ圧縮してもよい。

ただし完成版の設計では、未選択肢が蓄積し、戻り、履歴へ侵入する過程を体験させるため、20回以上の選択を想定する。
