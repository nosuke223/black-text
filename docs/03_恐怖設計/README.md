# 03_恐怖設計

## このフォルダの目的

BLACK TEXTで使用する「恐怖の骨格」を管理する。

生成AIに物語をすべて自由生成させるのではなく、ゲームとして一貫性が必要な要素をテンプレートとして定義し、AIにはその範囲内で舞台・描写・言い回しを生成させる。

---

## 基本構造

恐怖体験を、次の3つに分けて管理する。

```text
怪異テンプレート
    ＋
法則テンプレート
    ＋
試作シナリオ
```

### 怪異テンプレート

恐怖を引き起こす存在・現象の性質を定義する。

例：

- 何に反応するか
- どう近づくか
- 何をまねるか
- どのような痕跡を残すか
- どうすれば回避できる可能性があるか

怪異テンプレートには、特定の正解ルートや舞台を固定しすぎない。

---

### 法則テンプレート

プレイヤーが推測し、守ることで生存率を高められるゲームルールを定義する。

例：

- 三度目の呼びかけに反応してはいけない
- 同じ行動を繰り返してはいけない
- 自分と同じ声には返事をしてはいけない

法則は、怪異とは分けて再利用できる形にする。

---

### 試作シナリオ

怪異と法則を組み合わせて、1プレイ分の具体的な体験にする。

ここで初めて、

- 舞台
- 最初の3つの物
- シーン構成
- 手掛かり
- 内部状態
- クリア条件
- ゲームオーバー条件

を確定する。

## 使用手順

1. 使用する怪異テンプレートを選ぶ
2. 相性の良い法則テンプレートを選ぶ
3. 最初に提示する物と舞台を決める
4. 試作シナリオに落とし込む
5. 手掛かりと結果の因果関係を確認する
6. AIに任せる表現と、ゲーム側で固定する判定を分離する

---

## 現時点の方針

- MVPでは怪異・法則を大量に作らない
- まずは2〜3本の試作シナリオを作る
- 複数シナリオに共通して必要になった仕組みをゲームシステムへ昇格させる
- テンプレート同士の自動組み合わせは、個別シナリオが成立してから検討する
- AI生成によって法則や因果関係が壊れないことを最優先する
