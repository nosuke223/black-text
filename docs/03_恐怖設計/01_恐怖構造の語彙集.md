# 恐怖構造の語彙集

## 1. 目的

本ドキュメントは、試作シナリオで実際に使用された要素を観察し、怪異や法則より小さい恐怖構造の語彙として整理するための記録である。

目的は次の通り。

- 怪異や法則より小さい恐怖構造を整理する
- 新しいシナリオの種を考える際の発想補助にする
- 複数シナリオで再利用できる構造を見つける
- 将来の制約付きシナリオ生成へつなげる

これは正式な自動生成スキーマではない。

現在の怪異テンプレート、法則テンプレート、試作シナリオを置き換えるものでもない。

現段階では、観察できた構造と再利用候補を記録し、未検証の汎用性を決定事項として扱わない。

---

## 2. 用語の位置付け

恐怖構造は、次の関係で扱う。

```text
恐怖構造の語彙
        ↓
怪異テンプレート（検証済みの怪異プロファイル）
＋
法則テンプレート（検証済みの法則プロファイル）
        ↓
シナリオ契約
        ↓
ゲームシステムによる進行・判定
        ↓
AIによる文章・演出
```

本文では既存ファイル名に合わせて、「怪異テンプレート」「法則テンプレート」と呼ぶ。

現在の怪異テンプレートと法則テンプレートは、単一の最小部品ではない。

恐怖核、干渉チャネル、強化トリガー、法則骨格、対抗方法など、複数の小さな構造を意味のある形に束ねた、検証済みのプロファイルとして扱う。

恐怖構造の語彙を増やしても、プレイ実行時のAIへ法則や判定を委ねることにはならない。

使用する構造とその意味は、プレイ開始前にシナリオ契約として確定する。

---

## 3. 語彙の分類

以下の分類は、正式なスキーマではなく、試作シナリオを比較するための観察軸である。

| 分類 | 観察する内容 |
|---|---|
| 恐怖核 | プレイヤーが最終的に何を疑い、何を怖がるか |
| 干渉チャネル | 怪異や異常がプレイヤーへ届く媒体 |
| 干渉方法 | チャネルを通じて、何をどのように変えるか |
| 強化トリガー | プレイヤーのどの行動で干渉が強くなるか |
| 増幅方法 | 違和感がどのような段階を経て圧迫や侵入へ進むか |
| 法則骨格 | 舞台や表現を除いた、攻略上の中心ルール |
| 対抗方法 | プレイヤーが危険を避け、法則を確かめる行動 |
| 検証アイテム | 世界の状態や自分の履歴を確かめる重要アイテム |
| 手掛かり形式 | 法則や危険を示唆する情報の見せ方 |
| 状態悪化の示唆 | 内部状態を数値表示せずに伝える変化 |
| 最終局面型 | 過去の観察と法則を再利用させる終盤の問い |
| エンディング型 | 最終状態と選択の組み合わせによる結末の型 |

分類は必要に応じて追加・統合してよい。

ただし、分類を増やすこと自体を目的にせず、既存シナリオの具体的な怖さを説明できる範囲に留める。

## 5. 候補と正式部品の区別

恐怖構造の語彙は、次の2段階で管理する。

### 観察中の候補

- 1本の試作シナリオから見つかった構造
- 汎用性や別の組み合わせでの成立はまだ確認されていない
- 将来、別のシナリオで再利用できる可能性がある
- 元のシナリオに固有の舞台や表現を含む場合がある

候補として記録する段階では、無理に抽象化した名称へ置き換えない。

具体的な怖さと因果関係が分かる形で残す。

### 再利用確認済みの部品

- 複数の試作シナリオで使用された
- または別の怪異テンプレート、法則テンプレート、舞台との組み合わせでも成立することを確認できた
- 手掛かり、状態更新、最終局面まで一貫して利用できる
- 将来スキーマ化する候補になる

現段階では、001〜003の怪異テンプレートと法則テンプレートが一対一に近い形で使用されている。

そのため、個別の恐怖構造を再利用確認済みの部品と断定しすぎない。

文章やUIによる段階的な状態示唆など、複数シナリオで観察できる構造についても、ゲーム全体の共通原則なのか、組み合わせ可能な部品なのかを今後確認する。

---

## 6. 組み合わせ時の注意

- 組み合わせの大半は、必ずしも面白いシナリオにならない
- 意味が成立するだけでなく、恐怖としての必然性が必要である
- 法則と怪異の性質が因果関係によって結び付いている必要がある
- 法則を推測できる手掛かりを、舞台と出来事の中へ自然に配置できる必要がある
- 最終局面で、序盤から学んだ法則と対抗方法を再利用できる必要がある
- 小さな違反一回でクリア不能にせず、段階的な悪化を設計できる必要がある
- 自動的な完全ランダム組み合わせは行わない
- 将来は、相性の良い組み合わせと相性の悪い組み合わせを管理する可能性がある

組み合わせられないことは、語彙の失敗を意味しない。

特定の怪異テンプレートや法則テンプレートの中でのみ強く機能する構造も、その適用範囲を明記すればよい。

## 7. シナリオ契約

恐怖構造の語彙やテンプレートを、そのままプレイ実行時のAIへ渡して内容を決めさせない。

プレイ開始前に、今回の体験で変えてはいけない内容を「シナリオ契約」としてまとめる。

シナリオ契約に含める候補は次の通り。

- 今回の怪異
- 主法則
- 副法則
- 干渉チャネル
- 強化条件
- 対抗手段
- 重要アイテム
- 手掛かりの正準的な意味
- 内部状態
- 状態更新条件
- クリア条件
- ゲームオーバー条件
- 最終局面
- AIが変更してよいもの
- AIが変更してはいけないもの

シナリオ構築時にAIを使用する場合も、相性、因果関係、手掛かり、クリア可能性を確認し、プレイ開始前に内容を固定する。

プレイ実行時のAIは、シナリオ契約に沿って文章、舞台の細部、選択肢の自然な言い回し、状態に応じた演出を生成する。

法則、手掛かりの意味、内部状態の更新、生死判定は変更しない。

現段階では、シナリオ契約の正式なデータ形式やJSONスキーマは定義しない。

## 8. 新しいシナリオの種の例

以下は、恐怖構造の語彙を使った発想例である。

既存設定への追加や、完成シナリオとして確定するものではない。

### 種1：筆跡による模倣

```text
恐怖核：模倣
干渉チャネル：筆跡
強化トリガー：文章を書き写す
法則骨格：自分の筆跡で書かれた問いに返事を書いてはいけない
対抗方法：最初に残した署名や筆圧と照合する
```

自分の筆跡が、まだ書いていない返答を先回りして残す方向性。

### 種2：修正される館内放送

```text
恐怖核：記録と現在の不一致
干渉チャネル：館内放送と時刻表
強化トリガー：訂正放送に従って移動する
法則骨格：一度取り消された行き先を、再び目的地として選んではいけない
対抗方法：最初に印刷された時刻と現在地を照合する
```

訂正されるたびに、存在する出口や現在地の扱いが変わる方向性。

### 種3：届かなかった返答

```text
恐怖核：未実行行動の蓄積
干渉チャネル：未送信メッセージ
強化トリガー：過去の下書きを現在の返答として送信する
法則骨格：送らなかった返答を、後から同じ相手へ送ってはいけない
対抗方法：送信済み記録と端末外の控えを照合する
```

送らなかった言葉だけで、別の会話履歴が完成していく方向性。

これらの種を使用する場合も、怪異との因果関係、手掛かり、状態更新、最終局面まで成立するかを別途検証する。
