# BLACK TEXT 作業ルール

## プロジェクト概要

- BLACK TEXTは、生成AIを利用する選択式テキストホラーゲームである。
- ゲームとしての一貫性、攻略可能性、選択することの恐怖を優先する。

## 設計参照ルール

- コア体験に関する変更前に、`docs/01_コア体験.md`を確認する。
- ゲーム進行、選択、内部状態、AIとの役割分担に関する変更前に、`docs/02_ゲームデザイン.md`を確認する。
- 怪異、法則、手掛かり、シナリオに関する変更前に、`docs/03_恐怖設計/README.md`、`docs/03_恐怖設計/00_設計ルール.md`、`docs/03_恐怖設計/01_恐怖構造の語彙集.md`を確認する。
- 文書内の暫定目安、検討事項、確定仕様を区別する。

## 変更時の原則

- 指示された範囲を超えて、ファイルを勝手に変更しない。
- 一般方針を変更した場合は、影響を受ける雛形、既存シナリオ、関連文書を確認する。
- 範囲外の不整合を発見した場合は変更せず、完了報告に記載する。
- 複数ファイル間の用語、数値、状態名、設計方針の整合性を確認する。
- 未決事項を確定仕様として記載しない。
- 既存テンプレートや構成を、明示的な指示なく廃止・全面変更しない。
- プレイ実行時のAIへ、法則の変更、生死判定、手掛かりの意味の決定を任せない。

## 作業と確認

- 日本語MarkdownはUTF-8で保存し、文字化けがないことを確認する。
- 作業後に`git status --short`を確認する。
- `git diff`と`git diff --cached`を実行し、unstagedとstagedの両方の差分を確認する。
- `git diff --check`を実行する。
- 既存のテストや検証コマンドがある場合は実行する。
- 完了時に、変更ファイル、変更内容、影響範囲、未解決事項を報告する。
